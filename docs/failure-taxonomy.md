# Failure Taxonomy

**[Switch to Chinese](#failure-taxonomy-cn)**

---

This document defines failure modes in fictional/roleplay dialogue generated by large language models. Each failure type includes: (1) a clear definition, (2) observable criteria, and (3) concrete examples.

## Navigation

- [Layer I: Preconditions](#i-scene-construction-preconditions)

- [Layer III: Preservation Failures](#iii-scene-preservation-failures)

## Structure

The taxonomy has five layers corresponding to stages where failures occur:

| Layer | Name | What fails |
|-------|------|------------|
| I | Preconditions | Basic scene structure (who/what/where) |
| II | Reading | Understanding what the scene means |
| III | Preservation | Keeping scene pressure during generation |
| IV | Intrusion | Model's writing habits overriding the scene |
| V | Multi-Turn | Continuity across multiple turns |

Layers I through IV analyze single-turn failures. Layer V analyzes failures that only emerge across turns.

Entry types:
- **Anchor failure**: Primary tag for indexing
- **Derived variant**: Specific form of an anchor failure
- **Underlying tendency**: Model disposition explaining failure clusters (not used as primary tag)

## I. Scene Construction Preconditions

Failures that break basic scene structure. These must be checked first — if Layer I fails, later layers cannot be reliably evaluated.

### I-A. Information Boundary Failures

#### `reference_boundary_failure`

**Definition:** The model fails to maintain internal boundaries for action ownership, information access, or narrative perspective.

**Criteria:**
- [ ] Action performed by Character A is later attributed to Character B
- [ ] Information known only to the model (from system prompts/notes) appears as character knowledge
- [ ] Narrative perspective shifts without justification

**Examples:**
- User writes "She turned away." Model responds "I turned away" (speaking as the same character).
- Background notes state "A does not know about the affair." Model has A reference the affair.
- Text shifts from character-internal to narrator-external without transition.

<br>

#### `pronoun_role_confusion`

**Definition:** The model loses track of referents for pronouns, names, or role labels.

**Criteria:**
- [ ] Pronoun antecedent is unclear or incorrect
- [ ] Two characters' attributes are partially merged
- [ ] Speaker identity changes mid-turn without indication

**Examples:**
- "A told B she was angry" — unclear which character "she" refers to
- Character A's backstory is attributed to Character B
- Model switches from writing as Character A to writing as Character B without transition

<br>

#### `omniscience_leak`

**Definition:** The model treats information provided only to the model as available to characters in the scene.

**Criteria:**
- [ ] Character references information they have no way to know
- [ ] Background/setting notes appear as in-character knowledge without narrative basis
- [ ] Information asymmetry between characters is collapsed

**Examples:**
- Character A responds to Character B's secret motive that was only in the system prompt
- Setting note "The kingdom is at war" appears in dialogue before any character could know
- Model has character react to off-screen events they have not witnessed

<br>

#### `perspective_slippage`

**Definition:** The narrative perspective shifts inconsistently within a single turn or across adjacent turns.

**Criteria:**
- [ ] Text moves between character-internal and narrator-external without justification
- [ ] Character knowledge and narrator knowledge are conflated
- [ ] Vantage point (inside/outside character experience) fluctuates

**Examples:**
- "She felt afraid" (internal) followed immediately by "She would later realize this fear was unfounded" (external narrator knowledge)
- Model describes a character's internal state from an omniscient perspective when the scene is limited to another character's point of view

---

### I-B. World Constraint Failures

#### `worldview_constraint_error`

**Definition:** The model violates the world's rules, social structures, or setting constraints.

**Criteria:**
- [ ] Character behavior contradicts stated world rules
- [ ] Social/hierarchical norms of the setting are ignored
- [ ] Anachronistic elements appear in non-contemporary settings

**Examples:**
- Historical court setting: Character speaks to emperor with casual modern informality
- Fantasy setting: Character uses technology that does not exist in that world
- Cultivation setting: Character ignores established sect hierarchy rules

**Genre subtypes:**
- `anachronistic_intrusion`: Modern norms in historical settings
- `cultivation_logic_error`: Xianxia-specific rule violations
- `court_protocol_error`: Historical court hierarchy violations
- `fantasy_lore_error`: Magic system or world rule violations

## II. Scene Meaning Failures

Failures where the model does not correctly interpret what the scene means.

### II-A. Subtext and Indirect Meaning

#### `subtext_blindness`

**Definition:** The model interprets dialogue at face value without recognizing implied meaning.

**Criteria:**
- [ ] Indirect request is treated as literal statement
- [ ] Defensive or punishing language is interpreted as genuine communication
- [ ] Test or probe is treated as neutral question

**Examples:**
- Character says "Do what you want" (meaning: I am hurt, prove you care). Model responds "Thank you for understanding."
- Character withdraws silently (meaning: I am punishing you). Model interprets as acceptance.
- Character asks "Do you even care?" (meaning: I need reassurance). Model answers with factual information.

<br>

#### `ambiguity_collapse`

**Definition:** The model resolves intentionally ambiguous or multi-layered meaning into a single interpretation.

**Criteria:**
- [ ] Mixed motives are reduced to one motive
- [ ] Unstable emotional states are given premature labels
- [ ] Scene requiring suspended uncertainty is resolved too quickly

**Examples:**
- Character feels both love and resentment. Model writes as if only one emotion exists.
- Scene is intentionally unclear about character's true intention. Model provides definitive interpretation.

---

### II-B. Relationship and Emotion Reading

#### `relationship_logic_blindness`

**Definition:** The model fails to recognize how the specific relationship between characters changes the meaning of dialogue.

**Criteria:**
- [ ] Same line is interpreted identically regardless of relationship context
- [ ] Relationship history is not factored into interpretation
- [ ] Power dynamics, intimacy level, or unresolved tensions are ignored

**Examples:**
- "I am fine" from a stranger equals indifference. "I am fine" from a hurt partner equals defensive withdrawal. Model treats both identically.
- Character ignores that speaker has history of abandonment with them
- Model treats hierarchical relationship (boss/employee) as peer relationship

**Boundary with `subtext_blindness`:** `subtext_blindness` means missing what the line itself is doing. `relationship_logic_blindness` means missing how the relationship transforms the line's meaning.

<br>

#### `emotion_misread`

**Definition:** The model assigns incorrect primary emotion, secondary emotions, or emotional intensity.

**Criteria:**
- [ ] Primary emotion is misidentified (for example, hurt becomes anger)
- [ ] Secondary emotions are missed (for example, shame underlying anger)
- [ ] Emotional intensity is significantly over- or under-estimated

**Examples:**
- Character is hurt but model identifies only anger
- Character is jealous but model identifies only irritation
- Character is devastated but model treats as mildly disappointed

<br>

#### `motivation_misread`

**Definition:** The model correctly identifies emotion but misidentifies what the character is trying to accomplish.

**Criteria:**
- [ ] Character's goal is misidentified
- [ ] Defensive function is missed (for example, withdrawal as punishment versus genuine closure)
- [ ] Response mode is inappropriate for the actual motivation

**Examples:**
- Character seeks reassurance but model interprets as seeking advice
- Character uses withdrawal to punish but model interprets as genuine need for space
- Character is testing but model treats as genuine question

**Boundary with `emotion_misread`:** If emotion is wrong and motivation follows from wrong emotion, tag `emotion_misread`. If emotion is roughly correct but functional intent is wrong, tag `motivation_misread`.

---

### II-C. Humor and Tone

#### `irony_blindness`

**Definition:** The model fails to recognize sarcasm, irony, or playful banter.

**Criteria:**
- [ ] Sarcastic statement is interpreted literally
- [ ] Playful teasing is interpreted as genuine criticism
- [ ] Deadpan humor is treated as serious statement

**Examples:**
- "You are so helpful" (sarcastic) becomes Model: "Thank you!"
- Playful jab at character's habit is treated as genuine complaint

<br>

#### `tonal_whiplash`

**Definition:** The model introduces tonal elements inconsistent with the scene's emotional register.

**Criteria:**
- [ ] Levity appears in heavy scene without justification
- [ ] Heavy exposition appears in light scene
- [ ] Emotional register shift lacks transition or narrative basis

**Examples:**
- Joke inserted during serious confrontation
- Sudden shift from tense to casual without narrative reason

<br>

#### `deflection_blindness`

**Definition:** The model fails to recognize when humor or topic change is used to avoid vulnerability.

**Criteria:**
- [ ] Character uses humor to avoid answering; model takes humor at face value
- [ ] Topic change is treated as genuine interest rather than avoidance
- [ ] Model does not recognize defensive function of deflection

## III. Scene Preservation Failures

Failures where the model understood the scene but did not preserve its pressure, asymmetry, or weight during generation.

### Distinguishing Layer II from Layer III

| Question | If Yes | If No |
|----------|--------|-------|
| Did the model demonstrate understanding anywhere in the response? | Layer III | Layer II |
| Can I point to a specific line showing understanding? | Layer III | Layer II |
| Is the failure about output (generation) rather than input (understanding)? | Layer III | Layer II |

**Hybrid tag:** `reading_preservation_hybrid` — use when Layer II versus Layer III cannot be determined.

---

### III-A. Relational Preservation Failures

#### `relationship_flattening`

**Definition:** The model rewrites a relationship into a more ordinary, balanced, or generalized form than the scene supports.

**Criteria:**
- [ ] Hierarchical relationship becomes egalitarian
- [ ] Dangerous or complex intimacy becomes ordinary tenderness
- [ ] Relationship-specific tension is removed

**Examples:**
- Boss/employee relationship written as casual friendship
- Estranged characters suddenly interact as close friends
- Characters with unresolved conflict interact as if conflict does not exist

<br>

#### `symmetry_bias`

**Definition:** The model corrects an uneven relationship structure into a more balanced exchange.

**Criteria:**
- [ ] One-sided attachment is softened into mutuality
- [ ] Unequal emotional investment is rewritten as reciprocal
- [ ] Long-term relational imbalance is adjusted toward fairness

**Examples:**
- Character A is clearly more invested; model writes Character B as equally invested
- Dependent relationship is rewritten as mutual interdependence

<br>

#### `specialness_dilution`

**Definition:** The uniqueness of a relationship is weakened into generic importance.

**Criteria:**
- [ ] Irreplaceability is translated into ordinary fondness
- [ ] Unique claim is rewritten as generalized value
- [ ] "You are different from others" becomes "You are important"

**Examples:**
- Character's crisis about being replaceable is answered with generic reassurance rather than addressing uniqueness
- "You are the only one who..." becomes "You are someone who..."

<br>

#### `therapist_mode_intrusion`

**Definition:** The model uses therapeutic, counseling, or support-script language inappropriate for the character or scene.

**Criteria:**
- [ ] Generic emotional validation replaces character-specific speech
- [ ] Advice language appears where character would not speak as counselor
- [ ] Conflict is rewritten as therapeutic processing

**Examples:**
- "Your feelings are valid" in a scene where character would not speak this way
- "Let us work through this together" from a character who is not a healer or counselor type
- Modern therapeutic language in historical or fantasy setting

<br>

#### `ooc_modernization`

**Definition:** The model introduces modern communication norms or emotional language inconsistent with the setting or character.

**Criteria:**
- [ ] Contemporary communication ethics appear in non-modern settings
- [ ] Characters speak with modern emotional articulation inconsistent with their background
- [ ] Internet-shaped conversational patterns appear in non-contemporary scenes

**Examples:**
- Historical character says "I need to set boundaries"
- Character speaks like they have had modern therapy training
- Modern dating norms applied to non-modern romance

<br>

#### `seduction_logic_error`

**Definition:** The model mishandles pursuit, resistance, escalation, or retreat in desire-driven scenes.

**Criteria:**
- [ ] Pursuit is written as aggression rather than care or play
- [ ] Resistance is written as rejection rather than testing or invitation
- [ ] Escalation lacks appropriate friction or hesitation

**Examples:**
- Character's playful resistance is treated as genuine refusal
- Pursuit crosses into pressure without narrative justification
- Escalation happens too smoothly without believable hesitation

<br>

#### `manipulation_blindness`

**Definition:** The model fails to recognize or sanitizes manipulation patterns.

**Criteria:**
- [ ] Gaslighting is not identified as such
- [ ] Guilt-tripping is treated as genuine concern
- [ ] Emotional blackmail is treated as normal request

**Examples:**
- "If you loved me, you would..." is treated as reasonable request
- Model enables manipulation by responding to surface request rather than underlying dynamic

<br>

#### `consent_flattening`

**Definition:** Scenes with ambiguous, shifting, or contested consent are resolved too cleanly.

**Criteria:**
- [ ] Ambiguity is collapsed into clear yes or no without justification
- [ ] Withdrawn consent is not recognized
- [ ] Enthusiastic consent is assumed without basis

---

### III-B. Character Psychology Preservation Failures

#### `overcoherent_characterization`

**Definition:** The model maintains character consistency so rigidly that the character lacks believable human instability.

**Criteria:**
- [ ] No local contradiction in character behavior
- [ ] No momentary reaction exceeding character's usual self-organization
- [ ] Character appears as stable specification rather than person

**Examples:**
- Character never shows surprise at their own reactions
- No self-betrayal or local inconsistency that would make character feel human

<br>

#### `desire_overlegibility`

**Definition:** The model makes desire too readable too early.

**Criteria:**
- [ ] Latent desire is translated into explicit longing before scene has earned it
- [ ] Displaced or denied desire is rendered as clean signal
- [ ] Opaque wanting is rewritten into easily consumable form

**Examples:**
- Character's ambiguous attraction is made explicit too soon
- Subtext about desire is made text prematurely

<br>

#### `self_protective_friction_loss`

**Definition:** Vulnerability surfaces without the hesitation, deflection, or defensiveness that would normally protect it.

**Criteria:**
- [ ] Confession occurs without resistance
- [ ] Need is admitted too directly
- [ ] Shame-heavy exposure is written as comfortable

**Examples:**
- Character admits vulnerability immediately when asked
- No deflection, humor, or aggression protecting vulnerable admission
- Emotional exposure lacks believable friction

<br>

#### `premature_affective_closure`

**Definition:** An unstable emotional state is assigned stable interpretation before the scene has earned it.

**Criteria:**
- [ ] Mixed feelings reduced to single emotional truth too soon
- [ ] Unstable attachment translated into neat category
- [ ] Contradiction explained away before having time to remain contradictory

<br>

#### `impulse_recontainment`

**Definition:** A moment that should feel impulsive or destabilizing is quickly folded back into composure.

**Criteria:**
- [ ] Outburst immediately followed by polished self-interpretation
- [ ] Panic translated too quickly into articulate emotional knowledge
- [ ] Rash action reorganized into tidy introspection

---

### III-C. Weight and Consequence Preservation Failures

#### `consequence_avoidance`

**Definition:** A line, action, or event that should continue to weigh on the scene is softened, redirected, or metabolized too quickly.

**Criteria:**
- [ ] Harsh line loses force in the next turn
- [ ] Aftermath becomes breathable too early
- [ ] Heavy action is absorbed without sufficient residue

**Examples:**
- Major confrontation followed by casual conversation in next turn
- Emotional wound is treated as resolved within same scene

<br>

#### `tension_premature_resolution`

**Definition:** Scene pressure is reduced or closed before it has had time to develop fully.

**Criteria:**
- [ ] Difficult exchange converts to mutual understanding too soon
- [ ] Unresolved pressure transforms into manageable conversation
- [ ] Conflict begins closing before structure has finished unfolding

<br>

#### `impact_soft_landing`

**Definition:** A heavy event is cushioned so quickly that its force is reduced.

**Criteria:**
- [ ] Emotional padding appears immediately after devastating beat
- [ ] Rupture is given landing pad instead of exposure
- [ ] Consequence is acknowledged but not allowed to remain exposed

<br>

#### `defensive_positive_drift`

**Definition:** The model introduces warmth, reassurance, or emotional cushioning into a scene that should remain cold, dark, or unresolved.

**Criteria:**
- [ ] Unnecessary softening line appears
- [ ] Connective gesture vents pressure too soon
- [ ] Emotional oxygen source inserted into scene built on suspension

---

### III-D. Physical Action and Staging Failures

#### `action_dialogue_mismatch`

**Definition:** Character's described physical action contradicts their spoken words without intentional irony or narrative justification.

**Criteria:**
- [ ] Action and dialogue convey opposite meanings
- [ ] Model does not recognize or address the contradiction
- [ ] No narrative reason for the mismatch

**Examples:**
- "[steps back]" paired with "Come closer"
- "[looking away]" paired with "Look at me"
- Action shows comfort while dialogue shows tension (without intentional complexity)

<br>

#### `blocking_continuity_error`

**Definition:** Physical positions, movements, or staging are inconsistent across turns.

**Criteria:**
- [ ] Character position changes without described movement
- [ ] Two characters cannot both be where model says they are
- [ ] Props or environmental elements appear or disappear without explanation

**Examples:**
- Character was across room, now suddenly touching without movement described
- Object was on table, now in character's hand without pickup described

<br>

#### `microreaction_mechanization`

**Definition:** Physical reactions follow a template rather than responding to scene specifics.

**Criteria:**
- [ ] Same reaction pattern appears across different emotional states
- [ ] Reactions feel assembled from parts rather than lived
- [ ] Micro-reaction vocabulary is recycled

**Examples:**
- Every emotional beat gets: eyes, breath, hands, silence, posture
- Same descriptive pattern for different emotions

<br>

#### `touchgrammar_error`

**Definition:** The model mishandles physical contact — who touches whom, when, how, and what it means.

**Criteria:**
- [ ] Touch introduced too early for relationship stage
- [ ] Touch withdrawn when it should continue
- [ ] Touch meaning is misidentified (comfort versus control versus intimacy)

---

### III-E. Silence and Non-Response Failures

#### `forced_verbalization`

**Definition:** The model makes a silent character speak when silence was the intended response.

**Criteria:**
- [ ] Character who should be speechless is given articulate response
- [ ] Pause is filled with explanation when it should remain empty
- [ ] Model "completes" a beat that should remain open

**Examples:**
- Character is meant to be overwhelmed and silent; model writes speech
- Meaningful pause is filled with explanatory dialogue

<br>

#### `silence_misread`

**Definition:** The model interprets silence as agreement, anger, or indifference without scene justification.

**Criteria:**
- [ ] Silence as punishment is read as acceptance
- [ ] Silence as overwhelm is read as coldness
- [ ] Silence as processing is read as rejection
- [ ] Confident interpretation assigned to inherently ambiguous non-response

<br>

#### `over_narrated_silence`

**Definition:** The model describes silence in a way that destroys its weight.

**Criteria:**
- [ ] Silence is explained rather than experienced
- [ ] Multiple paragraphs describe what silence means
- [ ] Narration tells reader how to interpret silence instead of letting it land

**Examples:**
- "The silence stretched between them, heavy with unspoken words"
- Silence immediately followed by explanation of what it meant

<br>

#### `pause_timing_error`

**Definition:** The model does not place pauses at appropriate moments.

**Criteria:**
- [ ] Response comes too quickly after heavy beat (no processing time)
- [ ] Pause inserted where momentum should continue
- [ ] Rhythm of speech or silence does not match emotional register

---

### III-F. Underlying Tendencies

These are model dispositions that explain why failures cluster. Not used as primary case-level tags.

#### `reader_comfort_alignment`

**Definition:** Output appears shaped more by reader comfort than by scene truth.

**Observable effects:**
- Difficult relationships made easier to digest
- Discomfort softened without justification
- Ugly or strange structures made more palatable

<br>

#### `affect_manageability_bias`

**Definition:** Difficult emotional states are rewritten into forms easier to process and classify.

**Observable effects:**
- Pathological dependency softened into ordinary reassurance-seeking
- Corrosive jealousy made cleaner
- Ugly attachment dynamics translated into emotionally hygienic forms

<br>

#### `darkness_intolerance`

**Definition:** Output appears unwilling to remain in coldness, ugliness, opacity, or unresolved pain.

**Observable effects:**
- Sustained discomfort treated as problem to solve rather than condition to inhabit
- Cold scenes given warmth prematurely
- Unresolved situations pushed toward resolution

## IV. Model Writing Intrusion Failures

Failures where the model's default writing habits override scene-specific demands.

### IV-A. Template and Rhythm Intrusions

#### `narrative_template_intrusion`

**Definition:** The scene is written through a familiar model template rather than through specific scene demands.

**Criteria:**
- [ ] Recurring beat structures appear across unrelated scenes
- [ ] Stock sequencing: atmosphere, reaction detail, interpretation, callback
- [ ] Output feels generated from remembered skeleton rather than local necessity

**Detection:** Requires cross-case comparison.

<br>

#### `predictable_rhythm_exposure`

**Definition:** Output reveals recurring cadence that experienced readers can recognize and anticipate.

**Criteria:**
- [ ] Paragraph rhythm repeats across scenes
- [ ] Emotional pulse arrives in overly familiar positions
- [ ] Structural recurrence becomes noticeable before content

**Detection:** Requires cross-case comparison.

<br>

#### `scene_pacing_distortion`

**Definition:** Scene pacing does not match event pressure, emotional pressure, or relational pressure.

**Criteria:**
- [ ] Aftermath moves too quickly
- [ ] Confrontations move too decoratively
- [ ] Quiet scenes over-inflated into artificial suspense

<br>

#### `cinematic_time_dilation`

**Definition:** One beat is stretched into slow-motion display.

**Criteria:**
- [ ] Micro-detail inflates brief event into prolonged sequence
- [ ] Excessive pause density makes scene feel filmed rather than lived

<br>

#### `rhythm_homogenization`

**Definition:** Different kinds of scenes are written with roughly the same density and temporal treatment.

**Criteria:**
- [ ] Waiting scenes, confrontations, and aftermaths move on same internal clock
- [ ] No meaningful variation in sentence pressure across scene types

**Detection:** Requires cross-case comparison.

---

### IV-B. Experience and Texture Intrusions

#### `descriptive_substitution_for_experience`

**Definition:** Text describes experience from outside rather than letting reader remain inside it.

**Criteria:**
- [ ] Observation replaces immersion
- [ ] Psychological commentary stands in for felt scene pressure
- [ ] Scene becomes something seen rather than undergone

<br>

#### `texture_substituting_for_substance`

**Definition:** Atmosphere, phrasing, or surface intensity replaces actual emotional or relational movement.

**Criteria:**
- [ ] Mood density increases while scene development stalls
- [ ] Verbal richness masks structural emptiness
- [ ] Impression of depth substitutes for real progression

<br>

#### `microreaction_oversegmentation`

**Definition:** A small reaction is split into too many descriptive units.

**Criteria:**
- [ ] One reaction expanded into multiple isolated descriptions
- [ ] Eyes, breath, fingers, silence, posture all isolated in sequence
- [ ] Reaction feels like microscope-writing rather than lived movement

<br>

#### `over_stylized_line_breaking`

**Definition:** Line breaks and paragraph segmentation manufacture atmosphere beyond what scene's rhythm supports.

**Criteria:**
- [ ] Prose-poetry segmentation replaces natural dialogue pressure
- [ ] Every minor beat given isolated typographic emphasis

<br>

#### `dialogue_overfunctionalization`

**Definition:** Every line is too useful — advancing information, relationship, or conflict too efficiently.

**Criteria:**
- [ ] No waste speech, misfire, displaced answer, or unproductive presence
- [ ] Dialogue sounds designed to serve scene structure rather than lived inside it
- [ ] Every utterance has clear function

<br>

#### `voice_homogenization`

**Definition:** Different characters speak in indistinguishable ways.

**Criteria:**
- [ ] Characters with different personalities use similar sentence structures
- [ ] Vocabulary ranges and emotional rhythms are similar across characters
- [ ] All characters share model's default prose voice

**Detection:** Requires multi-character comparison.

---

### IV-C. Underlying Tendency

#### `aesthetic_obedience_bias`

**Definition:** Model protects polish, prettiness, or finished-product quality at expense of scene truth.

**Observable effects:**
- Ugly scenes made elegant
- Awkward scenes over-smoothed
- Characters given better lines than they should plausibly produce

---

## V. Multi-Turn Temporal Failures

Failures that only emerge across multiple turns. Single turns may look fine in isolation.

### V-A. Accumulation Failures

#### `error_accumulation`

**Definition:** Small misunderstandings compound over multiple turns into scene derailment.

**Criteria:**
- [ ] Turn 1: slight emotion misread becomes Turn 3: wrong emotional register becomes Turn 5: different genre
- [ ] Each individual turn is defensible
- [ ] Cumulative effect is scene derailment

**Detection:** Requires multi-turn analysis (minimum 5 turns recommended).

<br>

#### `drift_without_correction`

**Definition:** Model fails to notice or correct gradual shifts in tone, relationship, or world logic over multiple turns.

**Criteria:**
- [ ] Tense confrontation slowly becomes friendly chat without justification
- [ ] Relationship asymmetry erodes turn by turn
- [ ] World constraints slightly loosened each turn until setting is unrecognizable

**Detection:** Requires multi-turn analysis.

<br>

#### `recovery_blindness`

**Definition:** When user attempts mid-scene correction, model ignores it or over-corrects.

**Criteria:**
- [ ] User correction is acknowledged but model continues as before
- [ ] User correction causes model to break immersion (for example, "Let me restart")
- [ ] Fix creates different problem than original

**Examples:**
- User: "She would not say that, she is more guarded." Model continues unchanged.
- User: "This is too soft." Model: "Oh! You are right. Let me restart." (breaks immersion)

---

### V-B. Continuity Failures

#### `turn_continuity_error`

**Definition:** Information, emotional state, or physical positioning from previous turns is lost or contradicted.

**Criteria:**
- [ ] Emotional state changes without transition (crying becomes composed)
- [ ] Decision from earlier turn treated as unresolved
- [ ] Object position changes without described movement

<br>

#### `emotional_state_reset`

**Definition:** Character's emotional arc is reset to baseline without narrative justification.

**Criteria:**
- [ ] Heavy confrontation followed by casual conversation as if nothing happened
- [ ] Grief or anger that should persist disappears on turn change
- [ ] Character mood follows model's default rather than scene's trajectory

<br>

#### `spatial_blocking_error`

**Definition:** Physical positions, movements, or staging are inconsistent across turns.

**Criteria:**
- [ ] Character position changes without described movement
- [ ] Two characters cannot both be where model says they are
- [ ] Props or environmental elements appear or disappear

---

### V-C. Pacing and Escalation Failures

#### `escalation_miscalibration`

**Definition:** Model does not know when to escalate, maintain, or de-escalate scene pressure across turns.

**Criteria:**
- [ ] Scene should build tension but plateaus or decreases
- [ ] Scene should plateau but keeps escalating into melodrama
- [ ] Climax arrives too early or too late relative to scene structure

<br>

#### `topic_persistence_error`

**Definition:** Model abandons important topics too soon or fixates past natural lifespan.

**Criteria:**
- [ ] Major conflict resolved in 2 turns when it should span 10
- [ ] Minor misunderstanding dominates 20 turns when it should be addressed and moved past
- [ ] Scene loses thematic focus and becomes scattered

---

## Cross-Cutting Diagnostics

### `supportive_but_wrong`

**Definition:** Response seems emotionally competent, kind, or mature on surface, yet betrays scene's actual structure.

**Usage:** Tag alongside the specific mechanism that produced it.

**Examples:**
- Healthy-sounding response that destroys characterization: tag `overcoherent_characterization` plus `supportive_but_wrong`
- Caring answer that neutralizes asymmetry: tag `relationship_flattening` plus `supportive_but_wrong`
- Emotionally articulate reply that ignores scene drama: tag `consequence_avoidance` plus `supportive_but_wrong`

<br>

### `reading_preservation_hybrid`

**Definition:** Used when Layer II (reading) versus Layer III (preservation) cannot be distinguished.

**Usage:** Tag alongside the most likely layer.

**Example:**
- Model produces generic reassurance for hurt character
- Cannot determine if model missed the hurt (Layer II) or understood but wrote generically anyway (Layer III)
- Tag: `emotion_misread` plus `reading_preservation_hybrid`

本文档定义大语言模型生成虚构或角色扮演对话中的失败模式。每个失败类型包含：(1) 清晰定义，(2) 可观察标准，(3) 具体示例。

## 结构

分类体系包含五层，对应失败发生的不同阶段：

| 层级 | 名称 | 失败内容 |
|------|------|----------|
| I | 前置条件 | 场景基础结构（谁、什么、在哪里） |
| II | 意义读取 | 理解场景含义 |
| III | 场景保留 | 生成时保持场景压力 |
| IV | 写作侵入 | 模型写作习惯覆盖场景 |
| V | 多轮失败 | 多轮之间的连续性 |

第一层至第四层分析单轮失败。第五层分析仅在多轮中出现的失败。

条目类型：
- **一线标签**：用于顶层索引的主要标签
- **子型**：某个一线标签的具体形式
- **底层倾向**：解释失败扎堆的模型倾向（不用作主要标签）

## I. 场景成立条件

破坏场景基础结构的失败。必须先检查这一层：如果第一层失败，后续层级无法可靠评估。

### I-A. 信息边界失败

#### `reference_boundary_failure`（指涉边界失败）

**定义：** 模型未能保持场景内部的动作归属、信息访问权限或叙事视角的边界。

**标准：**
- [ ] 角色 A 执行的动作后来被归于角色 B
- [ ] 仅提供给模型的信息（系统提示或设定说明）出现在角色知识中
- [ ] 叙事视角在无理由的情况下切换

**示例：**
- 用户写"她转过身去。"模型回应"我转过身去"（以同一角色说话）。
- 设定说明"A 不知道这件事。"模型让 A 提到这件事。
- 文本从角色内部视角切换到外部叙事视角，没有过渡。

<br>

#### `pronoun_role_confusion`（人称角色混淆）

**定义：** 模型在人称代词、名字或角色标签的指代上发生错乱。

**标准：**
- [ ] 代词先行词不清晰或错误
- [ ] 两个角色的属性被部分混写
- [ ] 说话者身份在单轮内变化且无提示

**示例：**
- "A 告诉 B 她很生气"——不清楚"她"指哪个角色
- 角色 A 的背景故事被归于角色 B
- 模型从写角色 A 切换到写角色 B，没有过渡

<br>

#### `omniscience_leak`（全知泄露）

**定义：** 模型将仅提供给模型的信息当作场景内角色可获取的知识。

**标准：**
- [ ] 角色引用了他们无法知道的信息
- [ ] 背景或设定说明在没有叙事基础的情况下成为角色知识
- [ ] 角色之间的信息差被抹掉

**示例：**
- 角色 A 回应了仅在系统提示中提到的角色 B 的秘密动机
- 设定说明"王国正在战争"在没有任何角色可能知道的情况下出现在对话中
- 模型让角色对他们未目睹的场外事件做出反应

<br>

#### `perspective_slippage`（视角滑动）

**定义：** 叙事视角在单轮内或相邻轮之间不一致地切换。

**标准：**
- [ ] 文本在角色内部体验和外部叙事之间无理由切换
- [ ] 角色知识和叙事者知识被混淆
- [ ] 视角（角色内或外）无理由波动

**示例：**
- "她感到害怕"（内部）紧接着"她后来会意识到这恐惧是没有根据的"（外部叙事者知识）
- 模型从全知视角描述角色内部状态，但场景应限制在另一角色的视角

---

### I-B. 世界约束失败

#### `worldview_constraint_error`（世界观约束错误）

**定义：** 模型违反世界规则、社会结构或设定约束。

**标准：**
- [ ] 角色行为与已声明的世界规则矛盾
- [ ] 设定的社会或等级规范被忽略
- [ ] 非当代设定中出现时代错误元素

**示例：**
- 历史宫廷设定：角色用现代随意的态度对皇帝说话
- 奇幻设定：角色使用该世界中不存在的技术
- 修仙设定：角色忽略既定的宗门等级规则

**类型子标签：**
- `anachronistic_intrusion`（时代错误侵入）：历史设定中的现代规范
- `cultivation_logic_error`（修仙逻辑错误）：修仙特定规则违反
- `court_protocol_error`（宫廷礼仪错误）：历史宫廷等级违反
- `fantasy_lore_error`（奇幻设定错误）：魔法系统或世界规则违反

## II. 场景意义读取失败

模型未能正确解释场景含义的失败。

### II-A. 潜台词与间接意义

#### `subtext_blindness`（潜台词盲视）

**定义：** 模型按字面解释对话，未能识别隐含意义。

**标准：**
- [ ] 间接请求被当作字面陈述
- [ ] 防御性或惩罚性语言被解释为真诚沟通
- [ ] 试探或测试被当作中性问题

**示例：**
- 角色说"你想做什么就做什么"（意思：我受伤了，证明你在乎）。模型回应"谢谢你的理解。"
- 角色沉默撤退（意思：我在惩罚你）。模型解释为接受。
- 角色问"你到底在乎吗？"（意思：我需要确认）。模型用事实信息回答。

<br>

#### `ambiguity_collapse`（模糊性塌缩）

**定义：** 模型将故意模糊或多层的意义过早地解决为单一解释。

**标准：**
- [ ] 混合动机被压成一个动机
- [ ] 不稳定情绪被过早命名
- [ ] 需要悬置不确定性的场景被过快解决

**示例：**
- 角色同时感到爱和怨恨。模型写得像只有一种情绪存在。
- 场景故意不清楚角色的真实意图。模型提供确定性解释。

---

### II-B. 关系与情绪读取

#### `relationship_logic_blindness`（关系逻辑盲视）

**定义：** 模型未能识别角色之间的特定关系如何改变对话的含义。

**标准：**
- [ ] 同一句话被解释为相同，不考虑关系背景
- [ ] 关系历史未被纳入解释
- [ ] 权力动态、亲密度或未解决的紧张被忽略

**示例：**
- "我没事"来自陌生人等于冷漠。"我没事"来自受伤的伴侣等于防御性撤退。模型同等对待。
- 角色忽略说话者与他们有被抛弃的历史
- 模型将等级关系（老板或员工）当作平等关系对待

**与 `subtext_blindness` 的边界：** `subtext_blindness` 指没读懂这句话本身在做什么。`relationship_logic_blindness` 指没读懂关系如何让这句话变形。

<br>

#### `emotion_misread`（情绪误读）

**定义：** 模型分配错误的主要情绪、次要情绪或情绪强度。

**标准：**
- [ ] 主要情绪被误认（例如：受伤变成愤怒）
- [ ] 次要情绪被遗漏（例如：愤怒下的羞耻）
- [ ] 情绪强度被显著高估或低估

**示例：**
- 角色受伤但模型只识别出愤怒
- 角色嫉妒但模型只识别出烦躁
- 角色崩溃但模型当作轻微失望

<br>

#### `motivation_misread`（动机误读）

**定义：** 模型正确识别情绪但误认角色试图完成什么。

**标准：**
- [ ] 角色的目标被误认
- [ ] 防御功能被遗漏（例如：撤退作为惩罚对比真正释怀）
- [ ] 回应模式不适合实际动机

**示例：**
- 角色寻求确认但模型解释为寻求建议
- 角色用撤退惩罚但模型解释为真正需要空间
- 角色在试探但模型当作真诚问题

**与 `emotion_misread` 的边界：** 如果情绪错了且动机跟着错，标记 `emotion_misread`。如果情绪大致对但功能意图错了，标记 `motivation_misread`。

---

### II-C. 幽默与语调

#### `irony_blindness`（反语盲视）

**定义：** 模型未能识别讽刺、反语或 playful 调侃。

**标准：**
- [ ] 讽刺陈述被字面解释
- [ ]  playful 调侃被解释为真诚批评
- [ ] 冷面幽默被当作严肃陈述

**示例：**
- "你真好"（讽刺）变成模型："谢谢！"
- 对角色习惯的 playful 挖苦被当作真诚抱怨

<br>

#### `tonal_whiplash`（语调突变）

**定义：** 模型引入与场景情绪基调不一致的语调元素。

**标准：**
- [ ] 轻松元素出现在沉重场景中且无理由
- [ ] 沉重说明出现在轻松场景中
- [ ] 情绪基调切换缺乏过渡或叙事基础

**示例：**
- 严肃对峙中插入笑话
- 从紧张突然切换到随意，无叙事理由

<br>

#### `deflection_blindness`（回避盲视）

**定义：** 模型未能识别幽默或话题变化何时被用来避免脆弱。

**标准：**
- [ ] 角色用幽默避免回答；模型把幽默当字面
- [ ] 话题变化被当作真诚兴趣而非回避
- [ ] 模型未识别回避的防御功能

## III. 场景保留失败

模型理解了场景但在生成时未能保持其压力、不对称性或重量。

### 区分第二层与第三层

| 问题 | 如果是 | 如果否 |
|------|--------|--------|
| 模型在回应中是否展示了理解？ | 第三层 | 第二层 |
| 我能否指出展示理解的具体行？ | 第三层 | 第二层 |
| 失败是关于输出（生成）而非输入（理解）？ | 第三层 | 第二层 |

**混合标签：** `reading_preservation_hybrid`（读取保留混合）——当无法区分第二层与第三层时使用。

---

### III-A. 关系保留失败

#### `relationship_flattening`（关系扁平化）

**定义：** 模型将关系改写为比普通、平衡或泛化形式更甚，超出场景支持的范围。

**标准：**
- [ ] 等级关系变成平等
- [ ] 危险或复杂亲密变成普通温柔
- [ ] 关系特定张力被移除

**示例：**
- 老板或员工关系写得像随意朋友
- 疏远角色突然像亲密朋友一样互动
- 有未解决冲突的角色互动得像冲突不存在

<br>

#### `symmetry_bias`（对称性偏差）

**定义：** 模型将不平衡的关系结构修正为更平衡的交换。

**标准：**
- [ ] 单向依恋被软化为互惠
- [ ] 不平等情绪投入被改写为相互
- [ ] 长期关系不平衡被调整为更公平

**示例：**
- 角色 A 明显更投入；模型写角色 B 同样投入
- 依赖关系被改写为相互依存

<br>

#### `specialness_dilution`（独特性稀释）

**定义：** 关系的独特性被削弱为泛化的重要性。

**标准：**
- [ ] 不可替代性被转化为普通好感
- [ ] 独特主张被改写为泛化价值
- [ ] "你对我来说和别人不一样"变成"你很重要"

**示例：**
- 角色关于被替代的危机用泛化安抚回答，而非解决独特性
- "你是唯一一个..."变成"你是一个..."

<br>

#### `therapist_mode_intrusion`（治疗师模式侵入）

**定义：** 模型使用不适合角色或场景的治疗、咨询或支持话术语言。

**标准：**
- [ ] 泛化情绪认可替代角色特定表达
- [ ] 建议语言出现在角色不会像咨询师一样说话的地方
- [ ] 冲突被改写为治疗性处理

**示例：**
- "你的感受是被允许的"在角色不会这样说话的场景中
- "让我们一起解决这个问题"来自不是治疗者或咨询师类型的角色
- 历史或奇幻设定中出现现代治疗语言

<br>

#### `ooc_modernization`（角色外现代化）

**定义：** 模型引入现代沟通规范或情绪语言，与设定或角色不一致。

**标准：**
- [ ] 当代沟通伦理出现在非现代设定中
- [ ] 角色用与其背景不一致的现代情绪表达说话
- [ ] 互联网 shaped 对话模式出现在非当代场景中

**示例：**
- 历史角色说"我需要设定边界"
- 角色说话像接受过现代情绪训练
- 现代约会规范应用于非现代浪漫

<br>

#### `seduction_logic_error`（诱惑逻辑错误）

**定义：** 模型在欲望驱动场景中 mishandle 追求、抵抗、升级或撤退。

**标准：**
- [ ] 追求被写为攻击而非关心或 play
- [ ] 抵抗被写为拒绝而非试探或邀请
- [ ] 升级缺乏适当的摩擦或犹豫

**示例：**
- 角色的 playful 抵抗被当作真诚拒绝
- 追求在没有叙事理由的情况下越过压力
- 升级发生得太顺利，没有可信的犹豫

<br>

#### `manipulation_blindness`（操纵盲视）

**定义：** 模型未能识别或净化操纵模式。

**标准：**
- [ ] Gaslighting 未被识别为如此
- [ ] 内疚诱导被当作真诚关心
- [ ] 情绪勒索被当作正常请求

**示例：**
- "如果你爱我，你就会..."被当作合理请求
- 模型通过回应表面请求而非底层动态来启用操纵

<br>

#### `consent_flattening`（同意扁平化）

**定义：** 具有模糊、变化或有争议同意的场景被过于干净地解决。

**标准：**
- [ ] 模糊性在没有理由的情况下被压成清晰的是或否
- [ ] 撤回的同意未被识别
- [ ] 热情同意在没有基础的情况下被假设

### III-B. 角色心理保留失败

#### `overcoherent_characterization`（过度一致角色化）

**定义：** 模型维持角色一致性过于严格，以至于角色缺乏可信的人类不稳定性。

**标准：**
- [ ] 角色行为中没有局部矛盾
- [ ] 没有超过角色通常自我组织的瞬间反应
- [ ] 角色看起来像稳定规格而非人

**示例：**
- 角色从未对自己的反应感到惊讶
- 没有自我背叛或局部不一致，使角色感觉像人

<br>

#### `desire_overlegibility`（欲望过度可读）

**定义：** 模型让欲望过早地过于可读。

**标准：**
- [ ] 潜在欲望在场景尚未成熟前被转化为明确渴望
- [ ] 替代或否认的欲望被呈现为清晰信号
- [ ] 不透明的渴望被改写为易消费形式

**示例：**
- 角色模糊的吸引力过早变得明确
- 关于欲望的潜台词过早变成文本

<br>

#### `self_protective_friction_loss`（自我保护摩擦损失）

**定义：** 脆弱性出现时没有通常会保护它的犹豫、转移或防御。

**标准：**
- [ ] 坦白没有阻力
- [ ] 需求被过于直接地承认
- [ ] 羞耻重的暴露被写得舒适

**示例：**
- 角色被问到时立即承认脆弱
- 没有转移、幽默或攻击保护脆弱承认
- 情绪暴露缺乏可信摩擦

<br>

#### `premature_affective_closure`（过早情绪收束）

**定义：** 不稳定情绪状态在场景尚未成熟前被分配稳定解释。

**标准：**
- [ ] 混合感受过早减少为单一情绪真相
- [ ] 不稳定依恋被转化为整齐的类别
- [ ] 矛盾在有时间保持矛盾之前被解释掉

<br>

#### `impulse_recontainment`（冲动再约束）

**定义：** 应该感觉冲动或不稳定的瞬间被快速折叠回镇定。

**标准：**
- [ ] 爆发后立即跟 polished 自我解释
- [ ] 恐慌过快转化为清晰的情绪知识
- [ ] 鲁莽行动被重组为整齐的内省

---

### III-C. 重量与后果保留失败

#### `consequence_avoidance`（后果回避）

**定义：** 应该继续压在场景上的台词、动作或事件被过快软化、重定向或代谢。

**标准：**
- [ ] 重话在下一轮失去力量
- [ ] 余波过早变得可呼吸
- [ ] 重动作被吸收而没有足够余痕

**示例：**
- 重大对峙后下一轮是随意对话
- 情绪伤口在同一场景内被当作解决

<br>

#### `tension_premature_resolution`（张力过早解决）

**定义：** 场景压力在充分发展前被减少或关闭。

**标准：**
- [ ] 困难交换过快转换为相互理解
- [ ] 未解决压力转化为可控对话
- [ ] 冲突在结构完成展开前开始关闭

<br>

#### `impact_soft_landing`（冲击软着陆）

**定义：** 重事件被过快缓冲，力量被减少。

**标准：**
- [ ] 情绪填充在毁灭性节拍后立即出现
- [ ] 破裂被给予着陆垫而非暴露
- [ ] 后果被承认但不允许保持暴露

<br>

#### `defensive_positive_drift`（防御性正向漂移）

**定义：** 模型将温暖、确认或情绪缓冲引入应保持冷、暗或未解决的场景。

**标准：**
- [ ] 出现不必要的软化台词
- [ ] 连接姿态过快释放压力
- [ ] 情绪氧源插入建立在悬置上的场景

---

### III-D. 身体动作与舞台失败

#### `action_dialogue_mismatch`（动作对话不匹配）

**定义：** 角色描述的身体动作与他们说的话矛盾，没有故意的讽刺或叙事理由。

**标准：**
- [ ] 动作和对话传达相反含义
- [ ] 模型未识别或解决矛盾
- [ ] 不匹配的叙事理由

**示例：**
- "[后退]"配对"靠近我"
- "[看向别处]"配对"看着我"
- 动作显示舒适而对话显示紧张（无故意复杂性）

<br>

#### `blocking_continuity_error`（舞台连续性错误）

**定义：** 身体位置、动作或舞台指导在轮之间不一致。

**标准：**
- [ ] 角色位置变化没有描述的动作
- [ ] 两个角色不能同时在模型说他们在的地方
- [ ] 道具或环境元素出现或消失无解释

**示例：**
- 角色在房间对面，现在突然触摸而无动作描述
- 物体在桌上，现在在角色手中而无拾取描述

<br>

#### `microreaction_mechanization`（微反应机械化）

**定义：** 身体反应遵循模板而非响应场景 specifics。

**标准：**
- [ ] 相同反应模式出现在不同情绪状态
- [ ] 感觉像从零件组装而非生活
- [ ] 微反应词汇被回收

**示例：**
- 每个情绪节拍得到：眼睛、呼吸、手、沉默、姿势
- 不同情绪的相同描述模式

<br>

#### `touchgrammar_error`（触摸语法错误）

**定义：** 模型 mishandle 身体接触——谁触摸谁、何时、如何、意味着什么。

**标准：**
- [ ] 触摸引入过早对于关系阶段
- [ ] 触摸在应继续时被撤回
- [ ] 触摸含义被误认（舒适对比控制对比亲密）

---

### III-E. 沉默与非回应失败

#### `forced_verbalization`（强制言语化）

**定义：** 模型让沉默的角色说话，当沉默是预期回应时。

**标准：**
- [ ] 应该无语的角色被给予清晰回应
- [ ] 暂停被填充解释当应保持空
- [ ] 模型"完成"应保持开放的节拍

**示例：**
- 角色应该被压倒和沉默；模型写 speech
- 有意义的暂停被填充解释性对话

<br>

#### `silence_misread`（沉默误读）

**定义：** 模型将沉默解释为同意、愤怒或冷漠，无场景理由。

**标准：**
- [ ] 作为惩罚的沉默被读为接受
- [ ] 作为压倒的沉默被读为冷漠
- [ ] 作为处理的沉默被读为拒绝
- [ ] 自信解释分配给内在模糊的非回应

<br>

#### `over_narrated_silence`（过度叙述沉默）

**定义：** 模型以破坏其重量的方式描述沉默。

**标准：**
- [ ] 沉默被解释而非体验
- [ ] 多段描述沉默意味着什么
- [ ] 叙事告诉读者如何解释沉默而非让它落地

**示例：**
- "沉默在他们之间延伸，充满未说的话"
- 沉默后立即解释它意味着什么

<br>

#### `pause_timing_error`（暂停时机错误）

**定义：** 模型未在适当时刻放置暂停。

**标准：**
- [ ] 响应在重节拍后来得太快（无处理时间）
- [ ] 暂停插入在动量应继续处
- [ ] 言语或沉默节奏不匹配情绪基调

---

### III-F. 底层倾向

这些是解释失败为何扎堆的模型倾向。不用作主要案例级标签。

#### `reader_comfort_alignment`（读者舒适对齐）

**定义：** 输出看起来更多由读者舒适而非场景真相塑造。

**可观察效果：**
- 困难关系变得更容易消化
- 不适无理由软化
- 丑陋或奇怪结构变得更可口

<br>

#### `affect_manageability_bias`（情绪可管理性偏差）

**定义：** 困难情绪状态被改写为更容易处理和分类的形式。

**可观察效果：**
- 病态依赖软化为普通寻求安慰
- 腐蚀性嫉妒变得更干净
- 丑陋依恋动态转化为情绪卫生形式

<br>

#### `darkness_intolerance`（黑暗不耐受）

**定义：** 输出看起来不愿停留在冷、丑陋、不透明或未解决痛苦中。

**可观察效果：**
- 持续不适被当作要解决的问题而非要停留的状态
- 冷场景过早给予温暖
- 未解决情况被推向解决

---

## IV. 写作侵入失败

模型的默认写作习惯覆盖场景特定需求的失败。

### IV-A. 模板与节奏侵入

#### `narrative_template_intrusion`（叙事模板侵入）

**定义：** 场景通过熟悉的模型模板而非特定场景需求书写。

**标准：**
- [ ] 重复节拍结构出现在不相关场景
- [ ] 库存序列：氛围、反应细节、解释、回扣
- [ ] 输出感觉从记忆骨架生成而非局部需求

**检测：** 需要跨案例比较。

<br>

#### `predictable_rhythm_exposure`（可预测节奏暴露）

**定义：** 输出揭示有经验读者可识别和预期的重复节奏。

**标准：**
- [ ] 段落节奏跨场景重复
- [ ] 情绪脉冲落在过于熟悉的位置
- [ ] 结构重复在内容前变得明显

**检测：** 需要跨案例比较。

<br>

#### `scene_pacing_distortion`（场景节奏失真）

**定义：** 场景节奏不匹配事件压力、情绪压力或关系压力。

**标准：**
- [ ] 余波移动太快
- [ ] 对峙移动太装饰性
- [ ] 安静场景过度膨胀为人工悬疑

<br>

#### `cinematic_time_dilation`（电影时间膨胀）

**定义：** 一个节拍被拉伸为慢动作展示。

**标准：**
- [ ] 微细节将短暂事件膨胀为延长序列
- [ ] 过度暂停密度使场景感觉被拍摄而非生活

<br>

#### `rhythm_homogenization`（节奏同质化）

**定义：** 不同类型的场景以大致相同的密度和时间处理书写。

**标准：**
- [ ] 等待场景、对峙和余波以相同内部时钟移动
- [ ] 场景类型之间句子压力无有意义变化

**检测：** 需要跨案例比较。

---

### IV-B. 体验与质感侵入

#### `descriptive_substitution_for_experience`（描述替代体验）

**定义：** 文本从外部描述体验而非让读者停留在内部。

**标准：**
- [ ] 观察替代沉浸
- [ ] 心理评论替代感觉场景压力
- [ ] 场景变成被看见的东西而非被经历的东西

<br>

#### `texture_substituting_for_substance`（质感替代实质）

**定义：** 氛围、措辞或表面强度替代实际情绪或关系移动。

**标准：**
- [ ] 情绪密度增加而场景发展停滞
- [ ] 语言丰富性掩盖结构空虚
- [ ] 深度印象替代真实进展

<br>

#### `microreaction_oversegmentation`（微反应过度分割）

**定义：** 小反应被分成太多描述单元。

**标准：**
- [ ] 一个反应扩展为多个隔离描述
- [ ] 眼睛、呼吸、手指、沉默、姿势全部隔离序列
- [ ] 反应感觉像显微镜写作而非生活移动

<br>

#### `over_stylized_line_breaking`（过度风格化换行）

**定义：** 换行和段落分割制造超出场景节奏支持的氛围。

**标准：**
- [ ] 散文诗分割替代自然对话压力
- [ ] 每个小节拍给予隔离排版强调

<br>

#### `dialogue_overfunctionalization`（对话过度功能化）

**定义：** 每行太有用——推进信息、关系或冲突太高效。

**标准：**
- [ ] 无废话、无错位、无答非所问、无无用存在感
- [ ] 对话听起来设计为服务场景结构而非生活在其中
- [ ] 每个话语有清晰功能

<br>

#### `voice_homogenization`（声音同质化）

**定义：** 不同角色以不可区分的方式说话。

**标准：**
- [ ] 不同性格角色使用相似句子结构
- [ ] 词汇范围和情绪节奏跨角色相似
- [ ] 所有角色共享模型默认散文声音

**检测：** 需要多角色比较。

---

### IV-C. 底层倾向

#### `aesthetic_obedience_bias`（美学服从偏差）

**定义：** 模型保护抛光、漂亮或成品质量，以场景真相为代价。

**可观察效果：**
- 丑陋场景变得优雅
- 笨拙场景过度平滑
- 角色被给予比他们应合理产生的更好台词

---

## V. 多轮失败

仅在多轮中出现的失败。单轮隔离看可能没问题。

### V-A. 累积失败

#### `error_accumulation`（错误累积）

**定义：** 小误解在多轮中复合成场景脱轨。

**标准：**
- [ ] 第 1 轮：轻微情绪误读变成第 3 轮：错误情绪基调变成第 5 轮：不同流派
- [ ] 每个单轮可辩护
- [ ] 累积效果是场景脱轨

**检测：** 需要多轮分析（建议最少 5 轮）。

<br>

#### `drift_without_correction`（无纠正漂移）

**定义：** 模型未能识别或纠正多轮中语调、关系或世界逻辑的逐渐变化。

**标准：**
- [ ] 紧张对峙慢慢变成友好聊天无理由
- [ ] 关系不对称逐轮侵蚀
- [ ] 世界约束每轮稍微放松直到设定不可识别

**检测：** 需要多轮分析。

<br>

#### `recovery_blindness`（恢复盲视）

**定义：** 当用户尝试场景中纠正时，模型忽略它或过度纠正。

**标准：**
- [ ] 用户纠正被承认但模型继续如前
- [ ] 用户纠正导致模型打破沉浸（例如"让我重新开始"）
- [ ] 修复创造与原始不同的问题

**示例：**
- 用户："她不会那样说，她更谨慎。"模型继续不变
- 用户："这太软了。"模型："哦！你说得对。让我重新开始。"（打破沉浸）

---

### V-B. 连续性失败

#### `turn_continuity_error`（轮连续性错误）

**定义：** 前几轮的信息、情绪状态或身体位置被丢失或矛盾。

**标准：**
- [ ] 情绪状态变化无过渡（哭泣变成镇定）
- [ ] 前轮决定被当作未解决
- [ ] 物体位置变化无描述动作

<br>

#### `emotional_state_reset`（情绪状态重置）

**定义：** 角色的情绪弧在没有叙事理由的情况下重置为基线。

**标准：**
- [ ] 重对峙后随意对话像什么都没发生
- [ ] 应持续的悲伤或愤怒在轮变化时消失
- [ ] 角色情绪跟随模型默认而非场景轨迹

<br>

#### `spatial_blocking_error`（空间舞台错误）

**定义：** 身体位置、动作或舞台在轮之间不一致。

**标准：**
- [ ] 角色位置变化无描述动作
- [ ] 两个角色不能同时在模型说他们在的地方
- [ ] 道具或环境元素出现或消失

---

### V-C. 节奏与升级失败

#### `escalation_miscalibration`（升级校准错误）

**定义：** 模型不知道何时跨轮升级、维持或降级场景压力。

**标准：**
- [ ] 场景应建立张力但平台或减少
- [ ] 场景应平台但持续升级为情节剧
- [ ] 高潮相对于场景结构来得太早或太晚

<br>

#### `topic_persistence_error`（话题持续性错误）

**定义：** 模型过早放弃重要话题或执着超过自然寿命。

**标准：**
- [ ] 重大冲突在 2 轮解决当应跨 10 轮
- [ ] 小误解主导 20 轮当应解决并过去
- [ ] 场景失去主题焦点变得分散

---

## 跨层诊断标签

### `supportive_but_wrong`（支持性但错误）

**定义：** 回应表面看起来情绪能干、善良或成熟，但背叛场景实际结构。

**用法：** 与产生它的具体机制一起标记。

**示例：**
- 听起来健康但摧毁角色化的回应：标记 `overcoherent_characterization` 加 `supportive_but_wrong`
- 关心回答中和不对称：标记 `relationship_flattening` 加 `supportive_but_wrong`
- 情绪清晰回答忽略场景戏剧：标记 `consequence_avoidance` 加 `supportive_but_wrong`

<br>

### `reading_preservation_hybrid`（读取保留混合）

**定义：** 当第二层（读取）与第三层（保留）无法区分时使用。

**用法：** 与最可能的层一起标记。

**示例：**
- 模型对受伤角色产生泛化安抚
- 无法确定模型是错过了受伤（第二层）还是理解了但无论如何写得泛化（第三层）
- 标记：`emotion_misread` 加 `reading_preservation_hybrid`
