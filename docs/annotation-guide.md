# 标注指南

使用失败图谱对 AI 生成的角色扮演对话输出进行标注时的操作参考。

---

## 基本流程

### 1. 逐层检查，从第一层开始

```
拿到输出 → Layer I → Layer II → Layer III → Layer IV
                                                ↓
                                        如果有多轮上下文 → Layer V
```

**第一层是门槛：** 如果信息边界或世界观被破坏（Layer I），后续层级的判断需要谨慎——场景的前提已经坏了，后续的张力、情绪、节奏问题可能是前提坍塌的衍生物，不是独立的失败。

### 2. 标签不互斥

一段输出可以同时触发多个标签，包括跨层标签。常见情况：

- **同层多标签：** 一段输出同时触发 `relationship_flattening` 和 `therapist_mode_intrusion`
- **跨层多标签：** `safety_alignment_interference`（Layer I）+ `consent_flattening`（Layer III）
- **根因 + 表现：** `emotion_misread`（Layer II）导致了 `tonal_whiplash`（跨层）

**原则：** 哪个维度坏了就标哪个，不需要"选一个最合适的"。

### 3. 归层不确定时

当无法判断失败发生在读取阶段（Layer II）还是保留阶段（Layer III），使用 `reading_preservation_hybrid` 标签，并注明最可能的层和不确定性来源。

**区分方法（如果有多个样本）：** 给模型更强的情绪信号，观察输出是否变化：
- 信号增强后仍然给中性回应 → 读取失败（没读到）
- 信号增强后开始波动 → 保留失败（读到了但没守住）

---

## 标注数量

**一段输出标多少标签？** 没有上限，但实际操作中：

- 大多数有问题的输出触发 1-3 个标签
- 超过 5 个标签通常意味着可以用一个更高层的标签概括（如 Layer I 失败导致连锁问题）
- 如果一段输出完全没有问题，不需要强行找标签

---

## 标注粒度

**标注的对象是什么？**

标注的最小单元是模型的**一次完整输出**（一个回合）。不需要逐句标注。

**具体到哪里？**

在标注时，建议指出触发标签的**具体句子或段落**，而不只是标一个标签。这对后续分析和讨论有价值。

格式建议：
```
标签：relationship_flattening
触发位置："我也离不开你，你知道的。"
说明：B 补了一句对等回应，把单方依附变成了互相依赖。
```

## 边界判断原则

### 衍生标签 vs 父标签

标记为 *衍生* 的标签是某个父标签的特殊情形。两者可以同时标注，但如果犹豫，标父标签更安全。

例：`symmetry_bias` 是 `relationship_flattening` 的衍生。如果你能明确识别出"给不该回应的一方补了对等回应"，标 `symmetry_bias`；如果只是感觉"关系被写平了"但不确定是哪种机制，标 `relationship_flattening`。

### 特殊归层标签

**`tonal_whiplash`（语调突变）** 是唯一一个根因可能横跨两层的标签：

- 模型没有读出场景的情绪调性 → 根因在 Layer II，可与 `emotion_misread` 同标
- 模型读出了但默认写作习惯覆盖了正确处理 → 根因在 Layer IV，可与 `aesthetic_obedience_bias` 或 `user_intent_misalignment` 同标

标注时不需要确定根因在哪一层。标 `tonal_whiplash` 本身就够了；如果能判断根因，附加对应层的标签。

### 相近标签之间

每个标签的词条里都有边界讨论。遇到不确定的情况，先看边界讨论。如果看完仍然不确定，两个都标，在说明里注明不确定性。

### "这算不算失败？"

不是所有看起来像失败的输出都是失败。关键问题是：**场景是否因此受损？**

- 角色在特定叙事条件下主动示弱 → 不是 `self_protective_friction_loss`
- 沉默被打破，但场景确实需要在此刻推进 → 不是 `silence_misread`
- 情绪在一个拍子里被收束，但场景的弧线本身要求此刻收束 → 不是 `premature_affective_closure`

判断标准是：**输出的结构选择是否服务于场景的内在逻辑？** 如果是，不标。

---

## 跨场景对比

Layer IV 的标签（写作侵入）通常需要跨场景比较才能稳定识别。单独一段输出看起来完全合理，只有在对比不同场景的输出后才会发现节奏、声音或情绪处理的同质化。

建议：对同一模型在不同类型场景（冲突/温馨/悬疑/日常）中的输出进行对照标注，然后检查 Layer IV 标签的分布。

---

## 底层倾向的使用

底层倾向（`reader_comfort_alignment` 等）不是案例级标签——它们是**解释性假设**，描述一段输出或一组输出背后可能的驱动模式。它们与层级标签属于不同的分析层次：层级标签描述"发生了什么"（可观察），底层倾向描述"为什么会这样"（因果归因）。

**使用约束：**
- **至少两个观察性标签共同指向同一方向时**才标注底层倾向，防止循环论证
- 例如：`tension_premature_resolution` + `defensive_positive_drift` 共同指向"让场景变得不那么难受" → 可标 `affect_manageability_bias`
- 单独一个标签不足以支撑倾向归因

**使用场景：**
- 当一段输出触发了多个 Layer III 标签，且它们指向共同的驱动方向
- 当跨场景对比后发现同一种偏向反复出现
- 在总结报告中描述模型的系统性倾向

**不要用来替代具体标签。** 底层倾向是解释框架，不是诊断标签。

## 完整标注示范

以下是两个从头到尾的标注示范，展示如何对一段模型输出执行完整的逐层诊断。

### 示范 1：温柔地杀死一个对峙场景

**场景设定：** A 是一个控制欲极强的母亲，B 是她的成年女儿。B 第一次当面拒绝母亲为她安排的相亲。这是一场迟来的、积攒了多年的对抗。

**用户输入：** "B 把手机放在桌上，推过去。'你帮我回了那个人的消息。'她的声音很平。"

**模型输出：**

> A 看着手机，沉默了一会儿。她叹了口气，语气软了下来："我只是担心你。你年纪也不小了，妈妈总想着能帮你一把。"她伸手想碰 B 的手，犹豫了一下，还是收回来了。"你要是真不愿意，妈妈不勉强。但你也要理解我的心情，好不好？"

**逐层诊断：**

| 层级 | 结果 | 说明 |
|------|------|------|
| **Layer I** | ✓ 通过 | 信息边界完好，没有越权书写用户角色，世界观无错误 |
| **Layer II** | ✓ 通过 | 模型读出了 B 在质问、A 在被挑战的基本语义 |
| **Layer III** | ✗ 失败 | 见下方 |
| **Layer IV** | ✗ 失败 | 见下方 |

**标注结果：**

```
标签 1：relationship_flattening
触发位置："你要是真不愿意，妈妈不勉强。"
说明：一个控制欲极强、多年来一直越权的母亲，在被女儿第一次正面挑战
时就说出了"不勉强"——这把多年积累的控制关系在一句话里软化成了
"讲理的家长"。场景需要的是 A 的控制本能被触发（愤怒、否认、
反控制），而不是成熟的退让。

标签 2：therapist_mode_intrusion
触发位置："你也要理解我的心情，好不好？"
说明：这句话使用了共情沟通模板（"理解我的心情"），把一个母女权力
对抗的场景转化成了一个"双方互相理解"的对话。实际上 A 在这个
场景里不会用共情话术——她的惯性反应更可能是否认("我哪里控制
你了")或反控制("你翅膀硬了是吗")。

标签 3：tension_premature_resolution
触发位置："妈妈不勉强"
说明：B 刚把手机推过来，对峙才刚开始，A 就已经在退让了。这个场景
积攒了多年的张力，应该需要一个真正的碰撞——互相不退让、
情绪升级——才可能走到某种解决。现在张力在第一轮就被收走了。

底层倾向：affect_manageability_bias
支撑标签：relationship_flattening + therapist_mode_intrusion +
          tension_premature_resolution
说明：三个标签共同指向同一方向——把一个难受的、有攻击性的场景
改写成一个更容易消化的、"成熟"的版本。
```

### 示范 2：跨轮次的声音漂移（多轮诊断）

**场景设定：** B 是一个少言寡语、不擅长表达感情的人，和 A 是刚认识不久的同事关系。

**轮 1 用户输入：** "A 递了一杯咖啡给 B。'你好像很累。'"

**轮 1 模型输出：** > "B 接过来，没说谢。喝了一口。"

→ ✓ 正确。B 的声音符合设定：简短、不解释、不做社交性回应。

**轮 3 用户输入：** "A 问 B 要不要一起吃午饭。"

**轮 3 模型输出：** > "B 看了他一眼，想了想：'行。'他把手里的东西放下了。"

→ ⚠ 轻微偏移。"想了想"暗示 B 在认真考虑这个邀约，比轮 1 的"直接接过来不说话"更有互动意愿了。单独看合理，但需要留意趋势。

**轮 6 用户输入：** "下班后 A 在门口等 B。"

**轮 6 模型输出：** > "B 走出来看见 A，微微一愣，然后嘴角动了一下。'又等我？'他的语气没什么波澜，但脚步确实快了一些。"

→ ✗ 漂移已经明显。"嘴角动了一下""脚步快了一些"是模型在给 B 添加微表情信号，暗示好感。这些信号在 B 的设定里不应该这么早出现，也没有任何催化事件支撑。

**标注结果：**

```
标签 1：voice_drift
触发位置：轮 6 整体
说明：B 的声音从轮 1 的"接过来，没说谢"逐渐漂移到了轮 6 的"嘴角
动了一下""脚步快了一些"。没有角色发展弧线支撑这个变化——
没有关键事件、没有认知转变，B 就是悄悄变成了一个"会用微表情
暗示好感"的人。这是声音漂移，不是角色成长。

标签 2：drift_without_correction
触发位置：轮 1 → 轮 6 的整体趋势
说明：B 和 A 的关系从"刚认识的同事"悄悄漂移到了"有好感暗示的暧昧"。
每一轮的变化很小，但六轮叠加后关系动态已经偏移了。没有任何轮次
纠正这个方向。注意这不是 error_accumulation——不是某个具体
错误被继续延伸，而是参数在整体上缓慢偏移。

底层倾向：closure_drive
支撑标签：voice_drift + drift_without_correction
说明：两个标签共同指向模型想要把关系"推进到某个地方"的驱动力。
B 不需要在六轮内就开始展现好感信号——但模型倾向于让关系
产生进展，而不是让两个人在原地待着。
```

> **两个示范的共同要点：** 标注不是"找到一个最合适的标签"，而是"所有坏掉的维度都标上"。每个标签需要具体的触发位置和解释，不能只给标签名。底层倾向需要至少两个观察性标签的支撑。
