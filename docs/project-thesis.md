# Project Thesis

## 这个项目是什么

CN Failure Atlas 是一个面向虚构对话模型的失败模式分类体系。

项目的核心贡献是识别和整理那些有经验的读者经常能感觉到、但公开评测里很少被描述清楚的失败形式。

## 它要解决什么问题

公开的对话评测已经覆盖了流畅度、指令跟随、安全行为、广义人设一致性这些方面。但虚构和角色扮演对话暴露出另一类问题：

模型可以写得流畅、连贯、看起来也懂情绪、表面上也符合角色——但在场景层面上仍然是失败的。信息边界先坏了导致后续分析都站不住，情绪方向读对了但动机理解错了，关系被悄悄磨平了，欲望被太早翻译了，叙事在套模板而不是在接场景，后果被提前软化了。

所以项目的核心问题不只是"模型能不能正确理解和续写虚构场景"，还包括：

> 有哪些反复出现的结构性问题，在让模型的输出局部看似合理的同时，整体上让有经验的读者觉得是假的？

## 范围

**做什么：** 中文虚构对话和角色扮演风格的交互；信息边界和世界约束的维护；间接表达下的情绪识别和潜台词；关系逻辑驱动的场景理解；符合角色的续写和 OOC 分析；叙事结构和时间感的失真；后果处理和张力保持。

**不做什么：** 泛化的情绪支持质量评估；通用 chatbot 安全评测；全方位 roleplay 能力打分；单纯的人设知识正确性检查。

## 设计原则

**命名有价值。** 读者能感觉到但说不出名字的失败仍然是真实的失败。给它们命名和区分，本身就是这个项目想做的事。

**定义要清晰。** 每个失败类型都有清晰的定义、可观察的标准和具体示例，让"感觉不对"变成可以讨论的东西。

**好看不等于真实。** 打磨过的文本不一定是真实的场景。一致的角色不一定像一个人。修复不等于后果。这些区分是项目一直在做的事情。

## 和邻近工作的关系

已有工作覆盖了广义角色扮演评测、细粒度 OOC 检测、角色知识错误识别和角色模拟 agent 训练。本项目聚焦在一个更窄的缺口：模型能不能正确读懂虚构对话中的情绪、潜台词和关系逻辑，并在续写时不引入微妙的结构性失真。
