# Chinese Fictional Dialogue Failure Atlas

A structured atlas for failure modes in fictional / roleplay dialogue generated by large language models, centered on Chinese-language scenes.

This project names and organizes a class of failures that experienced readers often feel immediately but find hard to describe: not broken grammar, not obvious OOC, but the moment a scene stops being alive — when a relationship gets quietly rewritten into something flatter, when desire becomes too legible too early, when heavy things are softened before they land, or when the text starts moving through a recognizable template instead of actually happening.

## What This Is

Three layers:

1. **Failure Atlas** — names and distinguishes recurring failure modes, organized into five families
2. **Casebook** — demonstrates failures through concrete scenes, bad versions, reader drop points, and better alternatives
3. **Benchmark-Compatible Pilot Subset** — structured cases with YAML schema, Task A / B framing, prompt templates, and evaluation rubric

## Five Failure Families

| Family | What goes wrong |
|---|---|
| **I. Scene Reading** | The model misreads what the scene means — emotion, subtext, motive, relationship logic, or worldview constraints |
| **II. Relational Distortion** | The model rewrites the relationship into something flatter, more balanced, or more consumable |
| **III. Character Psychology** | The model turns people into overly coherent, legible, manageable psychological objects |
| **IV. Narrative / Temporal** | The model produces a polished artifact instead of a believable occurrence — templates, cinematic stretch, uniform rhythm |
| **V. Weight / Consequence** | The model won't let heavy things stay heavy — softening impact, inserting warmth, resolving tension too early |

Full taxonomy with all failure modes: [`docs/failure-taxonomy.md`](docs/failure-taxonomy.md)

# 中文虚构对话失败图谱

面向大语言模型虚构 / 角色扮演对话的失败模式图谱，以中文场景为中心。

这个项目要处理的不是一眼能看出的低级错误。更多时候，模型写出来的东西句子局部都还过得去，但整体越来越假——关系被偷偷写平了，欲望被过早翻译了，脆弱暴露时缺少该有的摩擦，重话落下以后马上被软化，或者整段场景在套模板而不是真的在发生。

这个仓库的工作就是把这些失败讲清楚、分清楚、展示出来。

## 结构

三层：

1. **失败图谱** — 命名和区分反复出现的失败模式，按五个家族组织
2. **案例集** — 用具体场景、坏版本、下头点和更好版本来展示这些失败
3. **Benchmark 兼容的 Pilot 子集** — 结构化 YAML 案例、Task A/B 设计、提示模板、评测 rubric

## 五个失败家族

| 家族 | 出了什么问题 |
|---|---|
| **I. 场景理解** | 模型没读懂场景——情绪、潜台词、动机、关系逻辑、世界观约束 |
| **II. 关系失真** | 模型把关系改写成更平、更对称、更好消费的版本 |
| **III. 角色心理** | 模型把人写得太整齐、太透明、太好管理 |
| **IV. 叙事 / 时间感** | 模型在生产打磨过的文本制品，而不是让场景真正发生 |
| **V. 重量 / 后果** | 模型不肯让重的东西继续重着 |

完整分类见 [`docs/failure-taxonomy.md`](docs/failure-taxonomy.md)
