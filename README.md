# Chinese Fictional Dialogue Failure Atlas

A structured taxonomy of failure modes in fictional and roleplay dialogue generated by large language models, focused on Chinese-language scenes.

## Problem Statement

When LLMs generate poor roleplay dialogue, the failure is often not grammatical errors or factual inconsistencies. Instead, the output feels structurally false despite being locally coherent. Common patterns include:

- Relationships rewritten as more balanced than the scene supports
- Vulnerability exposed without believable resistance
- Heavy emotional beats softened too quickly
- Scenes following familiar templates rather than responding to specific context

This project identifies, categorizes, and documents these failure modes with concrete examples.

## Project Structure

Three components:

| Component | Purpose | Location |
|-----------|---------|----------|
| **Failure Taxonomy** | Categorized failure modes with definitions and criteria | [`docs/failure-taxonomy.md`](docs/failure-taxonomy.md) |
| **Casebook** | Concrete scenes showing failures and corrected versions | `data/cases/` (in progress) |
| **Benchmark Subset** | YAML cases with Task A and B design and evaluation rubric | [`eval/`](eval/), [`data/cases/pilot/`](data/cases/pilot/) |

## Failure Layers

Five layers corresponding to stages where failures occur:

| Layer | Name | Definition |
|-------|------|------------|
| **I** | Preconditions | Basic scene structure fails (information boundaries, world rules) |
| **II** | Reading | Model does not understand scene meaning (subtext, emotion, relationship logic) |
| **III** | Preservation | Model understands but fails to maintain scene pressure during generation |
| **IV** | Intrusion | Model default writing habits override scene-specific demands |
| **V** | Multi-Turn | Failures emerging only across multiple turns (continuity, accumulation) |

Layers I through IV analyze single-turn failures. Layer V analyzes multi-turn failures.

Entry types:
- **Anchor failure**: Primary tag for indexing
- **Derived variant**: Specific form of an anchor failure
- **Underlying tendency**: Model disposition explaining failure clusters (not used as primary tag)

Full taxonomy: [`docs/failure-taxonomy.md`](docs/failure-taxonomy.md)

# 中文虚构对话失败图谱

面向大语言模型虚构和角色扮演对话的失败模式分类体系，以中文场景为中心。

## 问题定义

当大语言模型生成低质量角色扮演对话时，失败通常不是语法错误或事实不一致。相反，输出在局部连贯的情况下整体感觉结构虚假。常见模式包括：

- 关系被改写为比场景支持更平衡的形式
- 脆弱性暴露时没有可信的阻力
- 重情绪节拍被过快软化
- 场景遵循熟悉模板而非响应特定上下文

本项目识别、分类并用具体示例记录这些失败模式。

## 项目结构

三个组成部分：

| 组成部分 | 目的 | 位置 |
|----------|------|------|
| **失败分类** | 带定义和标准的分类失败模式 | [`docs/failure-taxonomy.md`](docs/failure-taxonomy.md) |
| **案例集** | 展示失败和修正版本的具体场景 | `data/cases/`（进行中） |
| **Benchmark 子集** | 带 Task A 和 B 设计和评估 rubric 的 YAML 案例 | [`eval/`](eval/)、[`data/cases/pilot/`](data/cases/pilot/) |

## 失败层级

五层对应失败发生的阶段：

| 层级 | 名称 | 定义 |
|------|------|------|
| **I** | 前置条件 | 场景基础结构失败（信息边界、世界规则） |
| **II** | 意义读取 | 模型未理解场景含义（潜台词、情绪、关系逻辑） |
| **III** | 场景保留 | 模型理解但在生成时未能保持场景压力 |
| **IV** | 写作侵入 | 模型默认写作习惯覆盖场景特定需求 |
| **V** | 多轮失败 | 仅在多轮中出现的失败（连续性、累积） |

第一层至第四层分析单轮失败。第五层分析多轮失败。

条目类型：
- **一线标签**：用于顶层索引的主要标签
- **子型**：某个一线标签的具体形式
- **底层倾向**：解释失败扎堆的模型倾向（不用作主要标签）

完整分类：[`docs/failure-taxonomy.md`](docs/failure-taxonomy.md)
